# TASK: nomnomcli v0.1 — agent-first calorie/nutrition CLI

## What we're building

`nomnomcli` — an open-source, agent-first CLI for precise calorie & nutrition tracking. The CLI is the single source of truth: all math happens in code against a bundled food database. An AI agent (Hermes, OpenClaw, etc.) is just the voice interface on top — it calls the CLI and narrates results. Never let the agent estimate nutrition itself.

Target user flow: human tells their agent "я съел борщ 300г и два куска хлеба" → agent runs `nomnom log --parse "борщ 300г, хлеб 2 куска"` → CLI resolves items against the bundled DB, computes kcal/macros deterministically, stores in local SQLite, returns JSON → agent narrates.

Install is one prompt: agent runs `curl -sL <install.sh> | sh`, which installs the CLI binary AND drops an agent skill file teaching the agent how to operate the CLI.

## Tech constraints

- **Python 3.11+**, package name `nomnomcli`, console script `nomnom`
- **Zero runtime deps beyond stdlib + requests** where possible (argparse/click — your call, but keep dep list minimal). recipe URL import may use `requests` + stdlib JSON-LD parsing; add `recipe-scrapers` ONLY if stdlib approach proves insufficient — document decision
- **SQLite** (stdlib sqlite3) for both the bundled food DB and the user DB
- **License: AGPL-3.0** (LICENSE file, headers not required)
- **English CLI interface** (commands/flags), but the food DB and RU-synonym layer must handle Russian food names ("гречка", "борщ", "творог")
- pytest for tests
- No network calls at runtime except: (a) recipe URL fetch on explicit `recipe add <url>`, (b) optional USDA API fallback for uncached foods (must work fully offline if no API key configured)

## Data strategy (hybrid, decided)

1. **Bundled mini-DB** (`nomnomcli/data/foods.sqlite`, committed to repo, target <2MB): ~300-500 most common foods with per-100g kcal/protein/fat/carbs, sourced from USDA FDC (SR Legacy / Foundation). Include a `scripts/build_mini_db.py` that regenerates it from USDA API (needs key, run by maintainer, not at install time).
2. **RU synonym map** (`nomnomcli/data/synonyms_ru.json`): ~200-300 entries mapping common Russian food/dish names → English canonical names in the DB ("гречка"→"buckwheat groats, roasted, cooked", "борщ"→"borscht"). Common prepared dishes (борщ, плов, оливье) get their own entries with typical kcal/100g.
3. **Optional USDA API fallback**: if `NOMNOM_USDA_KEY` env var is set, uncached lookups query the API and cache results into the user DB. Without the key: pure offline, mini-DB only.

## Commands (v0.1 scope)

- `nomnom log --parse "<free text>"` — resolve food items + quantities (support g/грамм, pieces/шт, ml), compute nutrition, store in user DB log, print JSON: `{items: [{name, grams, kcal, protein, fat, carbs, match_confidence}], totals: {...}}`. Non-zero exit + clear JSON error if an item can't be resolved (agent needs this to ask the human)
- `nomnom log --food "buckwheat" --grams 150` — direct form
- `nomnom stats today|week` — totals + per-meal breakdown, JSON + pretty table
- `nomnom recipe add <url>` — fetch page, parse schema.org Recipe JSON-LD, extract ingredients, resolve each against DB, store recipe with per-serving nutrition. `--servings N` override
- `nomnom recipe log <name> --portions 1.5` — log N portions of a stored recipe
- `nomnom search "<query>"` — search food DB, JSON list with kcal/100g
- All commands: `--json` for machine output (default pretty for humans). All output deterministic — no LLM anywhere in the CLI.

## Agent skill (deliverable!)

`skill/SKILL.md` — Hermes-format skill (YAML frontmatter: name `nomnomcli`, description with trigger phrases like "трекать калории", "посчитай калории", "что я сегодня ел"). Body teaches the agent:
- Install command
- Parse-then-confirm pattern: run `log --parse`, show the human the resolved items + grams, confirm before treating as final
- How to handle unresolved items (ask human for weight/clarification, retry with `--food --grams`)
- Recipe flow: `recipe add <url>` → `recipe log`
- Never estimate nutrition in the agent context — always call the CLI

## Install script

`install.sh` — POSIX sh: detects python3.11+, `pipx install git+https://github.com/maxjustships/nomnomcli` (fallback `pip install --user`), copies `skill/SKILL.md` to `~/.hermes/skills/nomnomcli/SKILL.md` if `~/.hermes` exists, prints next-steps. Idempotent.
